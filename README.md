# Битва за Респект — фронт

Турнирный сайт по Arc Raiders (стример Денис Блим). Next.js 16 (App Router, TS,
Turbopack, `output: 'standalone'`) + Tailwind v4 + React 19. Отдельный проект рядом
с Go-бэкендом ([arc-battles-back](https://github.com/anbolax1/arc-battles-back));
общаются только по HTTP через `/api`.

## Запуск (dev)
```bash
npm run dev        # http://localhost:3100
```
`/api/*` в dev проксируется на Go-бэкенд (`next.config.ts`; origin задаётся
`BACKEND_ORIGIN`, по умолчанию `http://127.0.0.1:8080`, локально — `:8081`,
см. `.env.local`). Для страниц с данными бэкенд должен быть запущен.

## Возможности
**Публичная часть:**
- Главная, расписание турниров (с пометкой «есть место»), рейтинг, архив.
- Страница турнира: участники/команды с никами, раунды, очки, победитель.
- Правила, профиль игрока, вход через Twitch.
- Регистрация: заявка в **общий пул** (Embark ID + удобные даты), без привязки к турниру.

**Кабинет организатора (`/admin`):**
- Расписание: создание турниров (1×1 / 2×2), управление составами, раундами, очками.
- Заявки: общий пул с поиском/счётчиком; постановка игрока в турнир прямо из заявки
  (заявка уходит из пула; при удалении участника — возвращается).
- Стартовые и бонусные задания, усложнения (справочники + раскладка по раундам).
- **Эфир**: VS-табло с суммарными очками, счётчики очков/штрафов/заданий с авто-сохранением,
  per-участник усложнения, перенос невыполненных бонусных заданий, авто-определение
  идущего раунда и победителя при завершении, защита завершённого турнира от правок.

**OBS-оверлей (`/overlay`):**
- Компактное табло (раунд, очки сторон, текущая команда, усложнение, пометка штрафа),
  авто-обновление по WebSocket. Ссылка для вставки в OBS с кнопкой копирования — в «Эфире».

## Структура
- `src/app/(site)` — публичные страницы; `src/app/admin` — кабинет; `src/app/overlay` — оверлей.
- `src/components/` — `ui/` (база: Button/Pill/Panel…), `domain/`, `admin/`, `overlay/`.
- `src/lib/` — `api.ts`/`server-api.ts` (клиент к `/api`), `queries.ts`, `types.ts`
  (зеркало Go-моделей), `ws.ts`, `format.ts`, `display.ts`, `auth.tsx`.

## Деплой (прод)
Split-топология: `brouhub.ru` (этот фронт, :3000) + `api.brouhub.ru` (Go API, :8080)
за nginx + Let's Encrypt. `NEXT_PUBLIC_API_BASE` вшивается при сборке (`.env.production`,
публичное значение). Сборку делать **локально** (на сервере 960 МБ RAM): `npm run build`,
затем standalone-комплект (`.next/standalone` + `.next/static` + `public`) на сервер,
user-сервис systemd под admin2. Полный раннбук и обновления — `../deploy/DEPLOY.md`.
